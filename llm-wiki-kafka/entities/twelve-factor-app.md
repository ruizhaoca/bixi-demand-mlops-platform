# Entity: The 12-Factor App

A methodology for building cloud-native, deployable apps — referenced in both Module I and II.

- 12 factors incl.: one codebase / many deploys, explicit dependencies, **config in environment**, backing services as attached resources, strict build/release/run separation, **stateless processes**, port binding, **scale out via process model**, disposability, dev/prod parity, **logs as event streams**, admin tasks as one-offs.
- Underpins the [[batch-stream-processing|service requirements]] (stateless, reusable) and [[cloud-native-architecture|cloud-native design]].
