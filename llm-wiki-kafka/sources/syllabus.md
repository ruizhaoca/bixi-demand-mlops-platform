## Key takeaways
- **Modules** (the wiki's spine): I Batch & Stream Processing ([[apache-spark|Spark]], [[apache-kafka|Kafka]]); II [[cloud-native-architecture|Cloud Native]] ([[docker|Docker]], [[kubernetes|Kubernetes]]); III Real-world ML in Production (SE best practices, safeguards); IV [[mlops-lifecycle|MLE/MLOps]]; V [[automl-and-tuning|AutoML & HPO]]; VI [[model-explainability|Explainability]] + [[fairness-bias-ml|Fairness/Bias]]; (optional) [[securing-ml-applications|Securing ML]] + special DL topics.
- **Tools**: Microsoft Azure, [[docker|Docker]], [[kubernetes|Kubernetes]], [[databricks|Databricks]] (Community Edition free), PyCharm, Python dev tools (pytest, black, mypy, flake8), open-source DS stack, H2O.ai AutoML, LIME/SHAP, EconML, **[[confluent-cloud|Confluent]] (Kafka)** — note Confluent needs a credit card (~$50/mo free credit for 3 months).
- **Evaluation**: Individual Assignment 1 = 15% ([[kafka-spark-assignment]]); Final Group Project = 70%; Class Participation = 15%.
- **Recommended texts**: Kleppmann *Designing Data-Intensive Applications* (ch. 5–7, 10–11 — the backbone of the data modules), Géron *Hands-On ML*, Hutter et al. *AutoML*, Molnar *Interpretable ML*, Barocas/Hardt/Narayanan *Fairness & ML*.
